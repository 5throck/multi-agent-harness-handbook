ワークショップ事前インストールチェックリスト

インストールスクリプトを実行する前にこのチェックリストを完了してください。以下の項目は自動化できないため、手動での対応が必要です。

1. サブスクリプションの確認（最優先）

ワークショップの実習で使用するツールには、有効なサブスクリプションが必要です。

ツール必要なプラン
Claude Code CLI（claude）Claude Pro または Max
ClaudeデスクトップアプリClaude Pro 以上
Antigravity CLI（agy）Gemini Advanced
AntigravityデスクトップアプリGemini Advanced
APIキーについて: APIキーは、サブスクリプションプランの範囲外で別途連携が必要な場合にのみ使用します。ワークショップの実習にはサブスクリプションモデルで十分ですので、APIキーの発行は本当に必要な場合にのみ行ってください。

2. アカウントの準備

  • Googleアカウント — Antigravity / Gemini Advanced の利用に必要
  • GitHubアカウント — gh auth login およびPR実習に必要
  • アカウントをお持ちでない場合は、第4章 §2-A 実習準備セクションの案内に従い、github.com/signupで事前に登録してください

3. システム要件

  • OS: Windows 11（Intel）/ macOS Sequoia 15 以上 Apple Silicon（Mシリーズ）/ Ubuntu 24.04 以上
  • 空きディスク容量: 5 GB 以上
  • 管理者 / sudo 権限を保有していること
  • Windows: スタート → 「PowerShell」を検索 → 右クリック → 「管理者として実行」（または Win + X → 「ターミナル（管理者）」）、UAC確認画面で「はい」をクリック
  • macOS / Linux: コマンドの先頭に sudo を付けて実行
  • 安定したインターネット接続（合計ダウンロード容量 約1〜2 GB）

4. Gitユーザー設定

gitインストール後、以下のコマンドでユーザー情報を登録してください。

git config --global user.name  "山田太郎"
git config --global user.email "you@example.com"
  • git config --global user.name の設定完了
  • git config --global user.email の設定完了

5. GitHub認証

gh auth login

案内に従って、ブラウザで認証を完了してください。

  • gh auth status 実行時に「Logged in」と表示されることを確認

6. プロジェクト環境設定（リポジトリのクローン後）

co-develop および co-consult variantを使用する場合:

cp .env.sample .env
# .envファイルを開いて必要な値を入力してください
  • .env.sample から .env ファイルを作成完了
  • 必要なAPIキー / 設定値の入力完了

co-develop Pythonプロジェクト — uv を使用（必須）:

uv sync          # pyproject.tomlに基づく依存関係のインストール
# または requirements.txt を使用する場合:
uv pip install -r requirements.txt
  • uv がインストールされ、使用可能であることを確認（uv --version）
  • uv によるPython依存関係のインストール完了
Dockerは任意です。運営者から別途案内があった場合のみインストールしてください。macOS/Linuxではインストールスクリプトに --docker フラグ、Windowsでは -Docker フラグを追加します。
  • （任意） Dockerのインストールとデーモンの起動確認

7. 最終環境確認（ワークショップ前に必ず実行）

プロジェクトのルートフォルダで以下のコマンドを実行してください。

bun setup-common.ts

出力テーブルのすべての項目が✅と表示されれば準備完了です。

ご不明な点がございましたら、ワークショップ開始前に運営者までご連絡ください。
