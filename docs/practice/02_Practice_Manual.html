4장 §1 · 하네스 기반 멀티 에이전트 팀 활용

Claude Code / Claude App / Antigravity CLI / Antigravity(Desktop) 네 도구에서 서브에이전트를 정의하고, 호출하고, 병렬·자동화 단위로 확장하는 방법을 나란히 정리한 레퍼런스  |  공식 문서: Claude Code — Subagents · Antigravity CLI — Getting Started · Antigravity CLI — Using · Antigravity — Agent Manager

대상 버전Claude Code 2026-07 릴리스 / Antigravity CLI (agy) 1.1.0+ / Antigravity 2.0
참조 문서 URLdocs.claude.com/en/docs/claude-code/sub-agents · antigravity.google/docs/cli-getting-started · antigravity.google/docs/cli-using · antigravity.google/docs/agent-manager
문서 수집일2026-07-14
버전 확인 방법Claude Code/App: claude --version · Antigravity CLI: agy --version 또는 세션 안에서 /usage · Antigravity Desktop: 앱 내 정보 화면
개정 시 확인 사항Claude Code Agent Teams와 Antigravity CLI/Agent Manager 모두 빠르게 변화하는 기능이라 플래그명·명령·UI 구성이 릴리스마다 바뀔 수 있으니 최신 릴리스 노트를 우선 확인
2026-06-18을 기점으로 Google은 개인(Free/Pro/Ultra) 사용자 대상 Gemini CLI 서비스를 종료했고, 그 자리를 Antigravity CLI(agy)가 대체했다. 이 장은 그 이전에는 Gemini CLI를 기준으로 작성됐으나, 현재 시점 기준으로 Antigravity CLI/Desktop으로 전면 갱신했다. 자세한 도구별 특성 비교는 도구 비교를 참고한다.

개요 — 전체 구조

하네스 기반 멀티 에이전트 팀 활용 (Desktop 앱을 기준으로, CLI는 동일 개념의 대안으로 병기)
├── 서브에이전트 정의하기
│   ├── Claude App / Claude Code    .claude/agents/*.md (YAML frontmatter, 사전 정의 — 둘 다 동일)
│   └── Antigravity(Desktop) / CLI  사전 정의 파일 없음 — 오케스트레이터가 목표를 보고 동적 생성
├── 서브에이전트 호출/위임하기
│   ├── Claude App / Claude Code    Agent(Task) 툴 — description / prompt / subagent_type
│   └── Antigravity(Desktop)/CLI    Agent Manager UI(Desktop) · /goal, /agent, /agents 패널(CLI)
├── 병렬 실행
│   ├── Claude App                  Agent Teams(in-process만)
│   ├── Claude Code                 Agent Teams(in-process/tmux) + 한 메시지에 여러 Agent 호출
│   ├── Antigravity Desktop         Agent Manager — Workspace별 에이전트 (Local/New Worktree Mode)
│   └── Antigravity CLI             세션 내부 병렬 서브에이전트 + 세션 간(터미널 여러 개) 병렬
└── 자동화/훅
    ├── Claude App                  훅 미발화 — 수동 실행 필요
    ├── Claude Code                 PostToolUse/TeammateIdle/TaskCompleted 훅 자동 발화
    ├── Antigravity Desktop         해당 없음 — GUI 중심
    └── Antigravity CLI             agy -p "프롬프트"(비대화형 CI용), agy inspect(설정 확인)
에이전트 팀 구성 예시

1. 서브에이전트 정의하기 — 사전 정의 vs 동적 생성

Anthropic 계열(Claude App/Claude Code)은 프로젝트 폴더 아래에 마크다운 파일 하나로 서브에이전트 하나를 미리 정의해 둔다. Google 계열(Antigravity Desktop/CLI)은 별도의 정의 파일이 없다 — 오케스트레이터가 주어진 목표를 보고 그때그때 서브에이전트를 만든다. 아래에서는 Claude App과 Antigravity(Desktop)를 기준으로 먼저 설명하고, 각각의 CLI 대응을 이어서 소개한다.

Claude 계열의 구체적인 정의·호출 방식은 §1-A Claude 중심에서, Antigravity 계열의 구체적인 방식은 §1-B Antigravity 중심에서 다룬다.

frontmatter 필드Claude App / Claude Code (.claude/agents)Antigravity (Desktop/CLI)
name 필수. subagent_type 값과 일치해야 호출 가능 해당 없음 — 정의 파일 자체가 없고, 오케스트레이터가 실행 시점에 역할·툴 범위를 스스로 판단해 서브에이전트를 생성한다
description 이 에이전트가 언제 쓰이는지 설명, 자동 트리거 판단에 참고됨
tools 사용 가능한 툴 목록, 생략 시 기본 세트 상속
model sonnet / opus / haiku 등 별칭 지정
Antigravity(Desktop/CLI)에는 "서브에이전트 정의 파일"이라는 개념 자체가 없다 — 초급 실습에서 Antigravity 쪽에 Claude App/Code의 reviewer.md와 똑같은 예제를 억지로 만들면 사실과 어긋난다. Antigravity의 "입문 실습"은 파일을 만드는 것이 아니라 목표를 주고 오케스트레이터가 알아서 작업을 쪼개는 과정을 관찰하는 것이다.

2. 서브에이전트 호출/위임하기

Anthropic 계열(Claude App/Claude Code)은 Agent/Task 툴로 별도 컨텍스트의 서브에이전트를 스폰한다. Google 계열(Antigravity Desktop/CLI)은 Agent Manager UI 또는 슬래시 커맨드로 에이전트에게 목표를 위임한다.

Claude 계열의 구체적인 호출 방식은 §1-A 섹션 2에서, Antigravity 계열의 구체적인 방식은 §1-B 섹션 2에서 다룬다.

3. 병렬 실행 — 4가지 도구 비교

네 도구 모두 병렬 실행을 지원하지만, 방식이 다르다. Claude 계열은 Agent Teams / 다중 Agent 호출, Antigravity 계열은 Agent Manager / 세션 간 병렬을 사용한다.

같은 디렉터리·Workspace에서 여러 에이전트를 동시에 돌리면 "인지적 중첩(cognitive overlap)"이 발생할 수 있다 — 한 에이전트가 같은 폴더에서 작업 중인 다른 에이전트의 결과를 자기 작업 맥락으로 착각(hallucination)하는 문제다. Workspace나 worktree로 에이전트를 분리하는 것이 권장 관행이며, 예외적으로 넘어갈 수 있는 사소한 문제가 아니다.

Claude 계열의 병렬 실행 방식은 §1-A 섹션 3에서, Antigravity 계열의 병렬 실행 방식은 §1-B 섹션 3에서 다룬다.

항목Antigravity (Desktop)Claude App (Desktop)Antigravity CLI (agy)Claude Code (CLI)
병렬 실행 단위 워크스페이스별 독립 에이전트 Agent Teams (in-process만) 세션 내부 + 세션 간(터미널 여러 개) 한 메시지 내 다중 Agent 호출 / Agent Teams(in-process + tmux)

4. 자동화/훅 — 무엇이 자동으로 발화하는가

Claude Code(CLI)는 이벤트 훅(PostToolUse/TeammateIdle/TaskCompleted)을 자동 발화하지만, Claude App과 Antigravity Desktop은 훅을 자동 발화하지 않는다. Antigravity CLI는 agy -p로 비대화형 실행, agy inspect로 설정 점검을 제공한다.

Claude 계열의 자동화/훅 세부는 §1-A 섹션 4에서, Antigravity 계열의 자동화/훅 세부는 §1-B 섹션 4에서 다룬다.

실무 팁: Antigravity에는 계산된 작업량(work done)에 기반한, 주기적으로 갱신되는 사용량 쿼터가 있다. 정확한 수치는 계정·플랜마다 다를 수 있으므로 여러 에이전트를 장시간 병렬로 돌리는 작업을 계획할 때는 이 점을 감안하는 편이 안전하다.

Agent Teams와 Agent Manager 모두 빠르게 발전 중인 기능이다 — 플래그명, 기본 동작, UI 구성이 릴리스마다 바뀔 수 있으므로 프로덕션 워크플로우에 의존하기 전에 최신 릴리스 노트를 확인하라.
