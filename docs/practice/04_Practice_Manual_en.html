1. Defining Subagents — Predefined vs. Dynamic Creation

The Anthropic family (Claude App/Claude Code) predefines each subagent as a single markdown file under the project folder. The Google family (Antigravity Desktop/CLI) has no separate definition file — the orchestrator creates subagents on the fly based on the given goal. Below, we first cover Claude App and Antigravity(Desktop) as the baseline, then introduce each one's CLI counterpart.

The concrete definition/invocation method for the Claude family is covered in §1-A Claude-Centered, and the concrete method for the Antigravity family is covered in §1-B Antigravity-Centered.

Frontmatter FieldClaude App / Claude Code (.claude/agents)Antigravity (Desktop/CLI)
name Required. Must match the subagent_type value to be invokable Not applicable — there is no definition file itself; the orchestrator determines the role and tool scope on its own at execution time to create the subagent
description Explains when this agent is used; referenced for automatic trigger decisions
tools List of available tools; inherits the default set if omitted
model Specify an alias such as sonnet / opus / haiku
Antigravity (Desktop/CLI) has no concept of a "subagent definition file" at all — forcing a beginner exercise that mirrors Claude App/Code's reviewer.md example onto Antigravity would misrepresent the facts. The "introductory exercise" for Antigravity isn't creating a file — it's giving it a goal and observing how the orchestrator breaks down the work on its own.

2. Invoking/Delegating Subagents

The Anthropic family (Claude App/Claude Code) spawns a subagent in a separate context using the Agent/Task tool. The Google family (Antigravity Desktop/CLI) delegates a goal to an agent via the Agent Manager UI or slash commands.

The concrete invocation method for the Claude family is covered in §1-A Section 2, and the concrete method for the Antigravity family is covered in §1-B Section 2.

3. Parallel Execution — Comparing 4 Tools

All four tools support parallel execution, but the approach differs. The Claude family uses Agent Teams / multiple Agent calls, while the Antigravity family uses Agent Manager / cross-session parallelism.

Running multiple agents simultaneously in the same directory/workspace can cause "cognitive overlap" — a problem where one agent mistakes the output of another agent working in the same folder as its own working context (hallucination). Isolating agents by workspace or worktree is the recommended practice, and this is not a minor issue that can be skipped as an exception.

The Claude family's parallel execution method is covered in §1-A Section 3, and the Antigravity family's parallel execution method is covered in §1-B Section 3.

ItemAntigravity (Desktop)Claude App (Desktop)Antigravity CLI (agy)Claude Code (CLI)
Parallel Execution Unit Independent agent per workspace Agent Teams (in-process only) Within a session + across sessions (multiple terminals) Multiple Agent calls within one message / Agent Teams (in-process + tmux)

4. Automation/Hooks — What Fires Automatically

Claude Code (CLI) automatically fires event hooks (PostToolUse/TeammateIdle/TaskCompleted), but Claude App and Antigravity Desktop do not fire hooks automatically. The Antigravity command-line interface (CLI) provides non-interactive execution via agy -p and configuration inspection via agy inspect.

Details on automation/hooks for the Claude family are covered in §1-A Section 4, and details for the Antigravity family are covered in §1-B Section 4.

Practical tip: Antigravity has a periodically refreshed usage quota based on calculated work done. Exact figures may vary by account/plan, so it's safer to keep this in mind when planning to run multiple agents in parallel for an extended period.

Both Agent Teams and Agent Manager are rapidly evolving features — flag names, default behavior, and UI layouts may change with each release, so check the latest release notes before relying on them in a production workflow.
