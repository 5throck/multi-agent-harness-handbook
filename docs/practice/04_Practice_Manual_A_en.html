Ch. 4 §1-A

Subagent Usage in Detail (Claude Focus)

A detailed document, limited to the Claude family, covering how to define, invoke, and scale subagents into parallel and automated units in Claude Code / Claude App  |  See the Ch. 4 §1 reference for common concepts, and Ch. 4 §1-B Antigravity Focus for the Antigravity family.

Target VersionClaude Code / Claude App 2026-07 release — see §1-B for the Antigravity counterpart
Reference Document URLdocs.claude.com/en/docs/claude-code/sub-agents
Document Collection Date2026-07-14
How to Check VersionClaude Code/App: claude --version

1. Defining Subagents

Claude App and Claude Code both predefine a single subagent using one Markdown file placed under the project folder. Claude App and Claude Code share the same files — both recognize .claude/agents/*.md in exactly the same way.

Claude App (Desktop) .claude/agents/*.md

Placing a Markdown file in the project root's .claude/agents/ folder is automatically recognized as a custom subagent in Claude App as well. The name becomes the identifier referenced later as subagent_type. The exact same file works exactly the same way in Claude Code (CLI) — the two share the same subagent definition format.

---
name: reviewer
description: Reviews text files for typos
  and logical gaps.
tools: Read, Grep
model: sonnet
---

You are acting as a reviewer. Read the given
file and report a list of improvements.

Claude Code (CLI) — Same .claude/agents/*.md as Claude App

It recognizes the exact same .claude/agents/*.md files written for Claude App, as-is. There is no difference in how subagents are defined between the Desktop App and the CLI.

Frontmatter FieldClaude App / Claude Code (.claude/agents)
name Required. Must match the subagent_type value to be callable
description Describes when this agent should be used; referenced for automatic trigger decisions
tools List of available tools; inherits the default set if omitted
model Specifies an alias such as sonnet / opus / haiku

2. Invoking/Delegating to Subagents

Both Claude App and Claude Code invoke the Agent tool (also referred to as the Task tool in documentation) to spin up a subagent with its own separate context.

Claude App (Desktop) Agent(Task) Tool

In Claude App as well, the main session invokes the Agent tool (also referred to as the Task tool in documentation) to spin up a subagent with its own separate context. Since the subagent cannot see the main session's conversation history, the prompt must self-sufficiently contain all the necessary background. Claude Code (CLI) uses the exact same Agent/Task tool — when you make a natural-language request in the chat window, it is internally invoked in a form like the one below.

Agent(
  description   = "Review authentication module",
  prompt         = "Review the session expiration
    handling logic in src/auth/login.ts and find bugs.",
  subagent_type  = "reviewer"
)

Claude Code (CLI) — Same Agent(Task) Tool as Claude App

This is the exact same Agent/Task tool invocation method as Claude App. Making a natural-language request in the terminal spawns a subagent in the same way.

3. Parallel Execution

The Claude family supports parallel execution in two ways.

Claude App (Desktop) Agent Teams — in-process

Turning on the experimental Agent Teams feature in Claude App also lets multiple teammates work in parallel. However, the Desktop App only supports teammateMode: in-process, and some keyboard navigation (such as Shift+Down) is restricted. Claude Code (CLI) supports the same in-process mode as well, and additionally the CLI lets you choose tmux mode, which uses tmux split panes.

Running multiple agents simultaneously in the same directory or workspace can cause "cognitive overlap" — a problem where one agent mistakes (hallucinates) the results of another agent working in the same folder as its own work context. Separating agents by working directory or worktree is the recommended practice.

Claude Code (CLI) — Multiple Agent Calls in One Message

When there are multiple independent tasks with no dependencies between them, you can place multiple Agent calls side by side within a single message to run them concurrently. Claude Code dispatches these calls in parallel.

Agent(description="Draft API documentation",
  prompt="...", subagent_type="writer")
Agent(description="Check test coverage",
  prompt="...", subagent_type="reviewer")
ItemClaude App (Desktop)Claude Code (CLI)
Parallel execution unit Agent Teams (in-process only) Multiple Agent calls in one message / Agent Teams (in-process + tmux)

4. Automation/Hooks

In the Claude family, the difference in automation and hooks lies between App and Code.

Claude App (Desktop) — Hooks Do Not Fire

Even if you define PostToolUse/TeammateIdle/TaskCompleted hooks in .claude/settings.json, they do not fire automatically in Claude App. Whatever the hook would have done (e.g., running an audit script) must be run manually after the session ends. In Claude Code (CLI), hooks in the same configuration file fire automatically in response to events — teams that need an automation gate should reasonably choose the command-line interface (CLI) at this point.

Claude Code (CLI) — Hooks Fire Automatically

The PostToolUse/TeammateIdle/TaskCompleted hooks in .claude/settings.json run automatically in response to events such as file saves or task completion.

{
  "hooks": {
    "PostToolUse": [
      { "matcher": "Write|Edit",
        "hooks": [{ "type": "command", "command": "bun scripts/audit.ts" }] }
    ]
  }
}
Agent Teams is a rapidly evolving feature — flag names, default behavior, and UI composition may change with each release, so check the latest release notes before relying on it for a production workflow.
