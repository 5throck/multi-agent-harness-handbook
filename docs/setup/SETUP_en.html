Workshop Environment Setup Guide

Complete the installation before the workshop begins — the goal is to be ready to jump straight into hands-on practice when the session starts, without any additional environment setup.

Setup Flow

1. OS updates
   └─ Windows: winget upgrade --all
   └─ macOS:   brew update && brew upgrade
   └─ Linux:   apt update && apt upgrade

2. Foundational tools
   └─ Windows: PowerShell 7+, Windows Terminal, Git (+ Git Bash)
   └─ macOS / Linux: curl, git

3. Runtimes
   └─ bun  (default runtime & package manager)
   └─ python3
   └─ uv   (Python package manager)

4. CLI tools
   └─ gh  (GitHub CLI)
   └─ claude  (Claude Code CLI)
   └─ agy  (Antigravity CLI)

5. Desktop apps
   └─ Google Chrome
   └─ Claude Desktop app
   └─ Antigravity Desktop app
   └─ Mark  (Markdown viewer)

6. Final verification ✅
   └─ bun setup-common.ts

Step 1 — Complete the Checklist

Before running the script, first read and complete the Pre-Installation Checklist.

Key items to confirm

  • Active subscription to Claude Pro/Max and Gemini Advanced (a subscription plan, not an API key)
  • A GitHub account created, ready for gh auth login
  • At least 5 GB of free disk space and confirmed administrator/sudo privileges

Step 2 — Run the OS-Specific Script

Open a terminal with administrator / sudo privileges and run the script for your OS.

macOS

bash setup-mac.sh
# Optional: also install WezTerm (GPU-accelerated terminal)
bash setup-mac.sh --wezterm
# Optional: also install Docker Desktop
bash setup-mac.sh --docker
# Both:
bash setup-mac.sh --wezterm --docker

Linux (Ubuntu / Debian)

bash setup-linux.sh
# Optional: also install WezTerm
bash setup-linux.sh --wezterm
# Optional: also install Docker
bash setup-linux.sh --docker
# Both:
bash setup-linux.sh --wezterm --docker

Windows

Run PowerShell as Administrator, then enter the command below.

# Default run (the script sets the Execution Policy automatically)
.\setup-windows.ps1

# Options:
.\setup-windows.ps1 -WSL2             # Also install WSL2 (restart required)
.\setup-windows.ps1 -WezTerm          # Also install WezTerm
.\setup-windows.ps1 -Docker           # Also install Docker Desktop
.\setup-windows.ps1 -Force            # Reinstall all tools, even already-installed ones
.\setup-windows.ps1 -WSL2 -WezTerm -Docker  # Install all optional extras
After running: close and reopen your terminal to apply PATH changes.

Troubleshooting

bun: command not found after installation

Restart your terminal. On Windows, bun is automatically registered in the User environment variables. macOS/Linux:

export PATH="$HOME/.bun/bin:$PATH"

claude: command not found after bun install -g

The script automatically retries with an npm fallback if bun fails. If the problem persists, check the path with bun pm ls -g and add it to your PATH.

agy: command not found after installation

The Antigravity CLI installer places the binary in ~/.local/bin or /usr/local/bin. Run source ~/.bashrc or restart your terminal.

Windows: "running scripts is disabled on this system"

The script automatically sets this to RemoteSigned. To change it manually:

Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
Windows: running without administrator privileges — the script displays a warning but continues. Some installs, such as winget, may fail, so it's recommended to rerun PowerShell as Administrator.

Checking installation logs

Windows script run logs are automatically saved to %USERPROFILE%\workshop-setup-logs\.

Run bun setup-common.ts, and once every item shows ✅, the workshop environment is ready.
