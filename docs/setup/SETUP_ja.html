ワークショップ環境セットアップガイド

ワークショップ開始前にインストールを完了させてください — セッション開始時に別途の環境構築なしですぐ実習に入れる状態を作ることが目標です。

セットアップの流れ

1. OSアップデート
   └─ Windows: winget upgrade --all
   └─ macOS:   brew update && brew upgrade
   └─ Linux:   apt update && apt upgrade

2. 基盤ツール
   └─ Windows: PowerShell 7+, Windows Terminal, Git (+ Git Bash)
   └─ macOS / Linux: curl, git

3. ランタイム
   └─ bun  (デフォルトランタイム & パッケージマネージャー)
   └─ python3
   └─ uv   (Pythonパッケージマネージャー)

4. CLIツール
   └─ gh  (GitHub CLI)
   └─ claude  (Claude Code CLI)
   └─ agy  (Antigravity CLI)

5. デスクトップアプリ
   └─ Google Chrome
   └─ Claudeデスクトップアプリ
   └─ Antigravityデスクトップアプリ
   └─ Mark  (Markdownビューア)

6. 最終確認 ✅
   └─ bun setup-common.ts

Step 1 — チェックリストの完了

スクリプトを実行する前に、事前インストールチェックリストを先に読んで完了させてください。

主な確認事項

  • Claude Pro/MaxおよびGemini Advancedの購読完了（APIキーではなく購読プラン）
  • GitHubアカウントの作成とgh auth loginの準備
  • 空きディスク容量5 GB以上、管理者/sudo権限の保有確認

Step 2 — OS別スクリプトの実行

ターミナルを管理者 / sudo権限で開き、ご自身のOSに合ったスクリプトを実行してください。

macOS

bash setup-mac.sh
# オプション: WezTerm (GPUアクセラレーションターミナル) も一緒にインストール
bash setup-mac.sh --wezterm
# オプション: Docker Desktopも一緒にインストール
bash setup-mac.sh --docker
# 両方:
bash setup-mac.sh --wezterm --docker

Linux (Ubuntu / Debian)

bash setup-linux.sh
# オプション: WezTermも一緒にインストール
bash setup-linux.sh --wezterm
# オプション: Dockerも一緒にインストール
bash setup-linux.sh --docker
# 両方:
bash setup-linux.sh --wezterm --docker

Windows

PowerShellを管理者権限で実行し、以下のコマンドを入力してください。

# 基本実行 (Execution Policyはスクリプトが自動で設定します)
.\setup-windows.ps1

# オプション:
.\setup-windows.ps1 -WSL2             # WSL2も一緒にインストール (再起動が必要)
.\setup-windows.ps1 -WezTerm          # WezTermも一緒にインストール
.\setup-windows.ps1 -Docker           # Docker Desktopも一緒にインストール
.\setup-windows.ps1 -Force            # 既にインストール済みのツールもすべて再インストール
.\setup-windows.ps1 -WSL2 -WezTerm -Docker  # オプションをすべてインストール
実行後: ターミナルを閉じて再度開き、PATHの変更を反映してください。

Step 3 — インストール確認

bun setup-common.ts

インストール済みツールの検証表を出力します。すべての項目が✅と表示されれば準備完了です。

トラブルシューティング

インストール後にbun: command not found

ターミナルを再起動してください。Windowsでは、bunはユーザー環境変数に自動的に登録されます。macOS/Linux:

export PATH="$HOME/.bun/bin:$PATH"

bun install -g後にclaude: command not found

スクリプトはbunが失敗した場合、npmフォールバックで自動的に再試行します。それでも問題が解決しない場合は、bun pm ls -gでパスを確認し、PATHに追加してください。

インストール後にagy: command not found

Antigravity CLIのインストールスクリプトは、バイナリを~/.local/binまたは/usr/local/binに保存します。source ~/.bashrcを実行するか、ターミナルを再起動してください。

Windows: 「このシステムではスクリプトの実行が無効になっています」

スクリプトが自動的にRemoteSignedに設定します。手動で変更する場合:

Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
Windows: 管理者権限なしで実行した場合 — スクリプトは警告を表示しますが、そのまま処理を続行します。wingetなど一部のインストールでエラーが発生する可能性があるため、PowerShellを管理者権限で再実行することを推奨します。

インストールログの確認

Windowsスクリプトの実行ログは%USERPROFILE%\workshop-setup-logs\に自動的に保存されます。

bun setup-common.tsを実行し、すべての項目が✅と表示されればワークショップの準備完了です。
