멀티 에이전트팀 하네스 엔지니어링 워크숍

프롬프트 엔지니어링에서 하네스 엔지니어링으로. Claude Code · Claude App · Antigravity · Antigravity CLI 네 도구를 넘나들며 멀티 에이전트팀을 직접 설계하고 운영하는 2일 실전 워크숍.

이 워크숍은 무엇인가? "프롬프트 엔지니어링만으로는 부족하다"는 문제의식에서 출발해, AI 에이전트에게 좁은 역할을 부여하고 오케스트레이터가 조율하는 멀티 에이전트팀의 설계 방법론을 배우고, Claude Code·Claude App·Antigravity·Antigravity CLI 네 도구에서 실제로 서브에이전트를 만들고 파이프라인을 구축해보는 이틀에 걸친 실전 워크숍이다.

1일차는 일반 사용자를 대상으로 멀티 에이전트 개념과 실무 도구 실습, 기존 variant 활용에 집중한다. 2일차는 IT전문가를 대상으로 아키텍처 심화, 도구 비교, 기업 운영 전략, 신규 variant 생성부터 캡스톤 프로젝트까지 진행한다.

🎯학습 목표

이 워크숍을 수료한 참가자는 다음을 할 수 있다.

LO-1
설명하네스 엔지니어링의 정의를 프롬프트·컨텍스트 엔지니어링과 비교하여 설명하고, 단일 에이전트의 세 가지 구조적 한계(컨텍스트 과부하·역할 충돌·병렬성 부재)를 든다.
LO-2
설계오케스트레이터·스페셜리스트·핸드오프 계약으로 이루어진 멀티 에이전트팀을 구성하고, Anthropic의 다섯 가지 오케스트레이션 패턴(체이닝·라우팅·병렬화·오케스트레이터-워커·평가자-최적화) 중 상황에 맞는 패턴을 선택한다.
LO-3
적용위험 작업 3단계 분류(읽기/쓰기/파괴적)와 최소 권한 원칙, 사람 승인 게이트, 격리, 감사 로그를 포함한 가드레일 체계를 서브에이전트에 설계한다.
LO-4
실행Claude Code·Claude App·Antigravity·Antigravity CLI 네 도구에서 서브에이전트를 정의하고 호출하며, 순차 파이프라인과 병렬 실행을 직접 구현한다.
LO-5
비교네 도구의 서브에이전트 정의 방식, teammateMode, 훅 지원, Agent Manager 차이를 실습 경험에 바탕으로 설명한다.
LO-6
판단기업 내 멀티 에이전트팀 운영의 두 가지 모델(서비스 티켓 모델 vs 개발 도구+variant 모델)을 비교하고, 자신의 조직에 적합한 모델과 그 이유를 설명한다.
LO-7
구현ai-workspace-standards 저장소의 L0→L1→L2 계층 구조와 핵심 파일 4종을 이해하고, 새로운 variant(co-marketing)를 프로토타입으로 만들어 승격 체크리스트를 작성한다.
LO-8
종합본인의 실제 반복 작업을 문제로 재정의하여, 문제 정의→가드레일 설계→서브에이전트 구현→검증·회고의 전체 사이클을 직접 설계하고 실행한다.

👥대상자

이 워크숍은 이틀 동안 다른 대상층을 대상으로 진행된다. 각 날짜에 맞춰 참가하거나, 이틀 모두 참석해 체계적으로 학습할 수 있다.

📅 1일차 — 일반 사용자

AI 도구를 업무에 활용하고 싶은 실무자를 대상으로 한다. 코딩 경험이 많지 않아도, CLI 기본 조작만 익숙하면 충분히 따라올 수 있다.

  • AI 코딩 도구를 처음 도입하거나 일상적으로 사용하는 실무자. Claude Code, GitHub Copilot, Cursor 등을 쓰고 있지만 단일 프롬프트로 한계를 느끼는 분
  • AI 에이전트 도입을 고민하는 팀 리더·기획자. 팀에 어떻게 에이전트를 배치할지 고민하는 분
  • AI 도구 교육을 담당하는 강사·컨설턴트. Claude Code와 Antigravity를 교육 자료로 다루는 분

📅 2일차 — IT전문가

1일차의 경험을 바탕으로, 아키텍처와 기술적 심화를 다루는 IT전문가 대상 세션이다.

  • AI 코딩 도구를 일상적으로 사용하는 소프트웨어 엔지니어. Claude Code, GitHub Copilot, Cursor 등을 쓰고 있지만 멀티 에이전트 아키텍처 설계 경험이 필요한 분
  • DevOps·플랫폼 엔지니어. 조직 단위 AI 워크스페이스 구축과 variant 관리가 업무에 포함된 분
  • AI 에이전트 도입을 설계하는 Tech Lead. 팀 전체에 어떻게 에이전트를 배치하고, 안전하게 운영할지 설계 책임이 있는 분
참고: 이 워크숍은 Claude Code와 Antigravity 계열 도구에 초점을 맞추고 있으며, 네 가지 도구의 설치와 CLI 기본 조작에 익숙한 것을 전제로 한다. CLI 경험이 없는 초보자는 설치 가이드의 사전 준비 단계를 충분히 연습하고 오는 것을 권장한다.

🕐강의 일정

이틀 총 소요 시간 약 9시간 15분 (휴식 제외). 1일차 휴식 2회, 2일차 휴식 2회(각 10~15분)를 포함하면 1일차 약 5시간 10분~5시간 40분, 2일차 약 4시간 45분~5시간 15분입니다.

일정 구성 원칙

1일차(일반 사용자)는 하네스 엔지니어링 개념과 가드레일, 실무 도구 실습에 집중하고, ai-workspace-standards를 가볍게 소개한 뒤 기존 variant 활용 실습을 충분히 진행합니다. 코딩 경험이 적은 참가자도 따라올 수 있도록 구성했습니다. 2일차(IT전문가)는 1일차의 경험을 바탕으로 ai-workspace-standards 아키텍처 심화, 도구 비교, 기업 운영 전략을 다루고, 신규 variant 생성과 캡스톤 프로젝트를 진행합니다.
