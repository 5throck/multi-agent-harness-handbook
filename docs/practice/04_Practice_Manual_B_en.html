Subagent Usage in Detail (Antigravity Focus)

A detailed document limited to the Antigravity family, covering how to define, invoke, and scale subagents into parallel/automated units in Antigravity CLI (agy) / Antigravity (Desktop)  |  common concepts are covered in the Chapter 4 §1 Reference, and the Claude family in Chapter 4 §1-A Claude Focus.

Target VersionAntigravity CLI (agy) 1.1.0+ / Antigravity 2.0 — see §1-A for the Claude counterpart
Reference Document URLantigravity.google/docs/cli-getting-started · antigravity.google/docs/cli-using · antigravity.google/docs/agent-manager

1. Defining Subagents

The Google family (Antigravity Desktop/CLI) has no separate definition file — the orchestrator creates subagents on the fly given a stated goal.

Antigravity (Desktop) Workspace-level — no separate definition file

Antigravity Desktop's Agent Manager handles agents at the Workspace level rather than as a set of pre-defined files. When a user gives a goal, the agent (orchestrator) assigned to that Workspace breaks down the subtasks and creates the subagents it needs on the spot. In other words, there is no concept of "preparing a file like reviewer.md in advance."

Antigravity CLI (agy) — Dynamically created by the orchestrator

Just like Antigravity Desktop, agy also has no pre-defined files — when given a goal, the orchestrator agent decides on its own how to break down the work and creates subagents on the spot to run them in parallel. Each subagent has its own independent context window, which keeps the main session's context from getting diluted.

Antigravity (Desktop/CLI) has no concept at all of a "subagent definition file" — forcing the same example used for Claude App/Code's reviewer.md onto the Antigravity side in a beginner exercise would be factually inaccurate. Antigravity's "introductory exercise" is not about creating files, but about giving a goal and observing the orchestrator break the work down on its own.

2. Invoking/Delegating to Subagents

The Antigravity family assigns goals to agents through the Agent Manager UI and slash commands.

Antigravity (Desktop) Agent Manager UI

In Antigravity Desktop, you pick a Workspace on the Agent Manager screen, spawn an agent to take on that Workspace (Local Mode or New Worktree Mode), and then give it a goal in natural language. Messages accumulate in the Inbox at points where approval or feedback is needed. This is less about "invoking a subagent" and more about "handing an agent a goal and watching its progress."

Antigravity CLI (agy) — /goal, /agent, /agents

The behavior of "dynamically spawning subagents as needed," the same thing Claude Code's Agent/Task tools do, is simply invoked through slash commands in agy (see §1-A Section 2). In an agy session, giving a goal with /goal runs it to completion without intermediate checks (without one, it pauses for confirmation at major steps), and /agent lets you hand a long-running task to the background while you continue with other prompts. /agents is a panel showing the list of running/completed/blocked subagents.

agy                      # Start a TUI session
/goal Review the notes.txt file and summarize the improvement points
/agents                  # List of running/completed subagents

For non-interactive automation, use agy -p "prompt" to run once and exit (for CI/automation). To check what settings/skills/plugins/hooks are loaded into a project, use agy inspect.

3. Parallel Execution

The Antigravity family supports parallel execution at two levels: the Workspace level and the session level.

Antigravity (Desktop) Assigning agents to multiple Workspaces via Agent Manager

Antigravity Desktop's Agent Manager is an interface operating at a level above the Editor, letting you manage multiple agents across multiple Workspaces at a glance. The default approach is interacting with the codebase through agents rather than writing code directly. In principle, it's recommended to assign only one agent per Workspace, but if needed, multiple agents can share one Workspace. When spawning an agent, you choose between Local Mode, which works directly in the active folder, and New Worktree Mode, which works in an isolated git worktree.

Running multiple agents simultaneously in the same directory/Workspace can cause "cognitive overlap" — a problem where one agent hallucinates by mistaking the results of another agent working in the same folder as its own work context. Separating agents by Workspace or worktree is the recommended practice, and this is not a minor issue that can be skipped as an exception.

Antigravity CLI (agy) — Parallelism within a session + across sessions

agy supports two levels of parallelism. Within a single session, the orchestrator runs multiple subagents at once, and across sessions, you can launch multiple terminals to run separate agy sessions independently in parallel — as long as they touch different files, this is safe. In environments where Antigravity Desktop's Agent Manager isn't available, this cross-session parallelism becomes the alternative.

ItemAntigravity (Desktop)Antigravity CLI (agy)
Parallel execution unit Independent agent per workspace Within a session + across sessions (multiple terminals)

4. Automation/Hooks

In the Antigravity family, automation is handled by the CLI's non-interactive mode and configuration-check commands.

Antigravity (Desktop) — Not applicable, GUI-centered

Antigravity Desktop is structured around managing approval and progress through a GUI, so it has no automatic firing mechanism equivalent to Claude Code's hooks. It's more realistic to move repetitive tasks that need automation over to Antigravity CLI.

Antigravity CLI (agy) — Non-interactive execution + configuration check

agy -p "prompt" is a non-interactive mode that runs once and exits, suitable for automation such as CI pipelines. agy inspect shows the configuration/skills/plugins/hooks loaded into a project — it doesn't correspond to Claude Code's automatic hook firing, but it plays the same role of confirming "what settings are actually applied right now."

agy -p "Draft a commit message"   # Non-interactive — runs once and exits
agy inspect
# → skills: 3 loaded, plugins: 1 loaded
# → hooks: none configured
# → config source: ~/.gemini/antigravity-cli/settings.json

Practical tip: Antigravity has a usage quota that refreshes periodically, based on computed work done. Exact figures may vary by account/plan, so it's safer to keep this in mind when planning to run multiple agents in parallel for extended periods.

Agent Manager is a rapidly evolving feature — flag names, default behavior, and UI layout can change with each release, so check the latest release notes before relying on it for production workflows.
